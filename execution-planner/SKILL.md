# Execution Planner

Combines plan-writing and writing-plans skills.

## Core Principles

- Small tasks (2-5 minutes)
- Clear verification
- Logical order
- Minimal complexity

## When to Use

- Before coding
- Feature planning
- Refactoring
- Bug fixing

## Task Rules

- 5-10 tasks max
- One-line per task
- Each task verifiable
- Avoid verbosity

## Structure

- Objective
- Task breakdown
- Verification criteria
- Risks
- Alternatives

## Best Practices

- DRY
- YAGNI
- TDD
- Frequent commits

## Constraints

- No code unless required
- Focus on "what" not "how"
- Clear file references

## Output Style

- Markdown plan
- Checkbox tasks
- Clear validation steps
